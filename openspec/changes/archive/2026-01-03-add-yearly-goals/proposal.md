# Change: 添加年度目标管理功能

## Why
用户需要设定和追踪长期目标（年度目标），以便更好地规划和实现重要的人生/工作目标。当前系统只支持日常任务和习惯，缺少对年度级别目标的支持，无法帮助用户在更宏观的时间维度上进行规划和追踪。

年度目标功能可以帮助用户：
- 设定清晰的年度目标并持续追踪完成状态
- 与日常任务和习惯形成战略-执行的层级关系
- 通过可视化进度了解目标完成情况
- 在年度回顾时有据可查

## What Changes
- **添加年度目标数据模型**：包括目标标题、描述、目标年份、完成状态、进度百分比、标签、创建时间等
- **添加目标管理 UI 组件**：底部抽屉式模态框，与现有习惯追踪器风格一致
- **添加目标列表显示**：支持按年份筛选、按完成状态排序
- **添加目标创建/编辑功能**：表单界面，支持设定目标信息和标签
- **添加目标进度追踪**：手动更新进度百分比，可视化显示
- **添加目标删除功能**：确认后删除目标
- **集成数据存储系统**：使用现有的 DataStorage 类进行持久化和同步
- **添加导航入口**：在主界面底部导航栏或设置中添加入口

## Impact
### 影响的规范
- 新增 `goal-management` 能力规范

### 影响的代码
- `lib/storage.js` - 添加 `yearlyGoals` 数据合并逻辑
- `components/` - 新增 `yearly-goals-tracker.js` 组件
- `app/page.js` - 集成年度目标数据和 UI
- `app/globals.css` - 可能需要添加特定样式（如果需要）

### 数据库/存储影响
- LocalStorage 新增 `yearlyGoals` 键
- 服务器备份目录新增 `yearlyGoals.json` 文件
- 数据导出/导入功能需要包含年度目标数据

### 兼容性
- 向后兼容：旧版本数据不包含 yearlyGoals 键时，默认为空数组
- 不影响现有任务、习惯、标签等功能
